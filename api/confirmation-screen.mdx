## Overview

The `ConfirmationScreen` widget allows users to review their captured photo, add a reflective narrative using the SHOWED methodology, and publish their post to the community feed.

**Location:** `lib/features/posts/presentation/screens/confirmation_screen.dart`

```dart
void _showShowedGuide(BuildContext context) {
  showModalBottomSheet(

              style: TextStyle(

                fontWeight: FontWeight.bold,

              'S - ¿Qué vemos aquí? (See)\n'
              'H - ¿Qué está pasando realmente? (Happening)\n'
              'O - ¿Cómo se relaciona con nuestras vidas? (Our lives)\n'
              'W - ¿Por qué existe esta situación? (Why)\n'
              'E / D - ¿Qué podemos hacer al respecto? (Empower / Do)',
              style: TextStyle(fontSize: 16, height: 1.5),
            ),
            // ...
          ],
        ),
      );
    },
  );
}
```

**SHOWED Framework:**
- **S (See)** - What do we see here?
- **H (Happening)** - What is really happening?
- **O (Our lives)** - How does this relate to our lives?
- **W (Why)** - Why does this situation exist?
- **E/D (Empower/Do)** - What can we do about it?

## Methods

### _onPublish()

Validates input and publishes the post.

```dart

```

**Flow:**
1. Validates that comment is not empty
2. Dismisses keyboard
3. Calls `PostProvider.publishPost()`
4. On success: pops navigation and shows success message
5. On error: shows error snackbar

#### Implementation

From `confirmation_screen.dart:24-58`:

```dart
Future<void> _onPublish() async {
  final comment = _commentController.text.trim();
  if (comment.isEmpty) {
    ScaffoldMessenger.of(context).showSnackBar(
      const SnackBar(content: Text('Por favor, añade una narrativa sobre tu foto.')),
    );
    return;
  }

  FocusScope.of(context).unfocus();

  final provider = Provider.of<PostProvider>(context, listen: false);

  try {
    await provider.publishPost(widget.imageFile, comment);
    
    if (!mounted) return;
    
    Navigator.pop(context);
    ScaffoldMessenger.of(context).showSnackBar(
      const SnackBar(
        content: Text('¡Voz compartida con éxito!'),
        backgroundColor: Colors.green,
      ),
    );
  } catch (e) {
    if (!mounted) return;
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(
        content: Text('Error al compartir: $e'),
        backgroundColor: Colors.redAccent,
      ),
    );
  }
}
```

## User Flow

1. User arrives from `HomeScreen` with captured photo
2. Photo is displayed with Hero animation
3. User can tap help icon to view SHOWED reflection guide
4. User writes narrative in text field
5. User taps "Compartir Voz" button
6. Validation ensures narrative is not empty
7. Loading overlay appears during upload
8. On success: returns to home screen with success message
9. On error: shows error message and stays on screen

## Error Handling

- Catches exceptions from `publishPost()`
- Checks `mounted` before showing UI feedback
- Displays user-friendly error messages in SnackBars
- Does not dismiss screen on error (allows retry)
