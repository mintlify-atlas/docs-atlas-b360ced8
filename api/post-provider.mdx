## Overview

The `PostProvider` class is a `ChangeNotifier` that manages post-related state and operations in the UI layer. It coordinates between the UI and the data repository, handling upload states and providing reactive data streams.

**Location:** `lib/features/posts/presentation/providers/post_provider.dart`

## Class Definition

```dart
class PostProvider extends ChangeNotifier {
  final PostRepository _repository;

  PostProvider(this._repository);
}
```

</ParamField>

## Methods

### publishPost()

Uploads an image and creates a new post with the provided comment.

```dart
Future<void> publishPost(File imageFile, String comment) async
```

<ParamField path="imageFile" type="File" required>
  The image file to upload
</ParamField>

<ParamField path="comment" type="String" required>
  User's narrative reflection about the photo
</ParamField>

#### Behavior

1. Sets `isUploading` to `true` and notifies listeners
2. Generates a unique filename using timestamp
3. Uploads the image to Supabase Storage
4. Creates a post record in the database
5. Sets `isUploading` to `false` and notifies listeners
6. Rethrows any errors for UI handling

#### Implementation

From `post_provider.dart:21-43`:

```dart
Future<void> publishPost(File imageFile, String comment) async {
  _isUploading = true;
  notifyListeners();

  try {
    // Generar un nombre único para la imagen
    final fileName = '${DateTime.now().millisecondsSinceEpoch}.jpg';
    
    // 1. Subir a Storage
    final imageUrl = await _repository.uploadImage(imageFile, fileName);
    
    // 2. Insertar en base de datos
    await _repository.createPost(imageUrl, comment);
    
  } catch (e) {
    // Manejo básico de error
    print('Error al publicar post: $e');
    rethrow;
  } finally {
    _isUploading = false;
    notifyListeners();
  }
}
```

<Warning>
This method rethrows exceptions. Always wrap calls in try-catch blocks to handle errors gracefully in the UI.
</Warning>

#### Complete Usage Example

From `confirmation_screen.dart:24-58`:

```dart
Future<void> _onPublish() async {
  final comment = _commentController.text.trim();
  if (comment.isEmpty) {
    ScaffoldMessenger.of(context).showSnackBar(
      const SnackBar(content: Text('Por favor, añade una narrativa sobre tu foto.')),
    );
    return;
  }

  FocusScope.of(context).unfocus();

  final provider = Provider.of<PostProvider>(context, listen: false);

  try {
    await provider.publishPost(widget.imageFile, comment);
    
    if (!mounted) return;
    
    Navigator.pop(context);
    ScaffoldMessenger.of(context).showSnackBar(
      const SnackBar(
        content: Text('¡Voz compartida con éxito!'),
        backgroundColor: Colors.green,
      ),
    );
  } catch (e) {
    if (!mounted) return;
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(
        content: Text('Error al compartir: $e'),
        backgroundColor: Colors.redAccent,
      ),
    );
  }
}
```

## State Management Pattern

The provider follows the ChangeNotifier pattern:

- Extends `ChangeNotifier` from Flutter
- Calls `notifyListeners()` when state changes
- UI widgets use `context.watch<PostProvider>()` to rebuild on state changes
- One-time operations use `context.read<PostProvider>()` to avoid unnecessary rebuilds

## Provider Setup

Typically registered at app initialization:

```dart
ChangeNotifierProvider(
  create: (context) => PostProvider(PostRepository()),
  child: MyApp(),
)
```
