## Overview

The `PostRepository` class handles all data operations for posts, including image uploads to Supabase Storage and CRUD operations on the posts table. It provides both Future-based and Stream-based methods for data access.

**Location:** `lib/features/posts/data/repositories/post_repository.dart`

## Class Definition

```dart
class PostRepository {
  final SupabaseClient _supabase;

  PostRepository({SupabaseClient? supabaseClient})
      : _supabase = supabaseClient ?? Supabase.instance.client;
}
```

<ParamField path="supabaseClient" type="SupabaseClient" optional>
  Optional Supabase client instance. Defaults to `Supabase.instance.client` if not provided. Useful for testing.
</ParamField>

## Methods

### uploadImage()

Uploads an image file to Supabase Storage and returns the public URL.

```dart
Future<String> uploadImage(File file, String fileName) async
```

<ParamField path="file" type="File" required>
  The image file to upload
</ParamField>

<ParamField path="fileName" type="String" required>
  Unique filename for the image (e.g., timestamp-based)
</ParamField>

<ResponseField name="returns" type="Future<String>">
  Public URL of the uploaded image in Supabase Storage
</ResponseField>

#### Implementation

From `post_repository.dart:13-25`:

```dart
Future<String> uploadImage(File file, String fileName) async {
  final storagePath = 'public/$fileName';
  
  // Subir el archivo al bucket "photos"
  await _supabase.storage.from('photos').upload(storagePath, file);
  
  // Obtener la URL pública para luego guardarla en la BD
  final String publicUrl = _supabase.storage
      .from('photos')
      .getPublicUrl(storagePath);
      
  return publicUrl;
}
```

#### Storage Configuration

- **Bucket name:** `photos`
- **Path pattern:** `public/{fileName}`
- Returns the public URL that can be accessed without authentication

### createPost()

Inserts a new post entry into the Supabase database.

```dart
Future<void> createPost(String imageUrl, String comment) async
```

<ParamField path="imageUrl" type="String" required>
  Public URL of the uploaded image from Storage
</ParamField>

<ParamField path="comment" type="String" required>
  User's narrative reflection about the photo
</ParamField>

#### Implementation

From `post_repository.dart:28-35`:

```dart
Future<void> createPost(String imageUrl, String comment) async {
  final Map<String, dynamic> data = {
    'image_url': imageUrl,
    'comment': comment,
  };
  
  await _supabase.from('posts').insert(data);
}
```

<Note>
The `id` and `created_at` fields are automatically generated by Supabase, so they are not included in the insert operation.
</Note>

### fetchPosts()

Retrieves all posts from the database ordered by creation date (newest first).

```dart
Future<List<Post>> fetchPosts() async
```

<ResponseField name="returns" type="Future<List<Post>>">
  List of Post objects ordered from most recent to oldest
</ResponseField>

#### Implementation

From `post_repository.dart:38-45`:

```dart
Future<List<Post>> fetchPosts() async {
  final response = await _supabase
      .from('posts')
      .select()
      .order('created_at', ascending: false);
  
  return (response as List).map((postMap) => Post.fromJson(postMap)).toList();
}
```

#### Query Details

- Selects all columns from the `posts` table
- Orders by `created_at` in descending order (newest first)
- Converts JSON response to `Post` objects

### fetchPostsStream()

Provides a real-time stream of posts that automatically updates when data changes.

```dart
Stream<List<Post>> fetchPostsStream()
```

<ResponseField name="returns" type="Stream<List<Post>>">
  A stream that emits updated post lists whenever the database changes
</ResponseField>

#### Implementation

From `post_repository.dart:48-54`:

```dart
Stream<List<Post>> fetchPostsStream() {
  return _supabase
      .from('posts')
      .stream(primaryKey: ['id'])
      .order('created_at', ascending: false)
      .map((maps) => maps.map((map) => Post.fromJson(map)).toList());
}
```

#### Stream Details

- Uses Supabase real-time subscriptions
- Primary key is `id` for tracking changes
- Orders by `created_at` descending
- Automatically emits new data when posts are added, updated, or deleted

<Tip>
Use `fetchPostsStream()` for the main feed UI to get real-time updates. Use `fetchPosts()` for one-time operations like data export.
</Tip>

## Complete Workflow Example

From `post_provider.dart:21-43`:

```dart
Future<void> publishPost(File imageFile, String comment) async {
  _isUploading = true;
  notifyListeners();

  try {
    // Generate unique filename
    final fileName = '${DateTime.now().millisecondsSinceEpoch}.jpg';
    
    // 1. Upload to Storage
    final imageUrl = await _repository.uploadImage(imageFile, fileName);
    
    // 2. Insert into database
    await _repository.createPost(imageUrl, comment);
    
  } catch (e) {
    print('Error al publicar post: $e');
    rethrow;
  } finally {
    _isUploading = false;
    notifyListeners();
  }
}
```

## Error Handling

All methods throw exceptions on failure. The repository does not catch or handle errors internally - error handling should be done at the provider or UI level.

Common errors:
- Network connectivity issues
- Storage quota exceeded
- Invalid file format
- Database constraint violations
