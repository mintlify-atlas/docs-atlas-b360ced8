## Flutter SDK Installation

<Tabs>
  <Tab title="macOS">
    <Steps>
      <Step title="Download Flutter">
        Download the latest stable Flutter SDK:

        ```bash
        cd ~/development
        wget https://storage.googleapis.com/flutter_infra_release/releases/stable/macos/flutter_macos_3.16.0-stable.zip
        unzip flutter_macos_3.16.0-stable.zip
        ```
      </Step>

      <Step title="Add to PATH">
        Add Flutter to your PATH in `~/.zshrc` or `~/.bash_profile`:

        ```bash
        export PATH="$PATH:$HOME/development/flutter/bin"
        ```

        Reload your shell:

        ```bash
        source ~/.zshrc
        ```
      </Step>

      <Step title="Verify Installation">
        ```bash
        flutter doctor
        ```

        This will check for any missing dependencies.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Windows">
    <Steps>
      <Step title="Download Flutter">
        1. Download the Flutter SDK from [flutter.dev](https://flutter.dev/docs/get-started/install/windows)
        2. Extract the zip file to `C:\src\flutter`
      </Step>

      <Step title="Update PATH">
        Add Flutter to your PATH:

        1. Search for "Environment Variables" in Windows
        2. Edit the `Path` variable
        3. Add `C:\src\flutter\bin`
        4. Click OK
      </Step>

      <Step title="Verify Installation">
        Open a new Command Prompt and run:

        ```bash
        flutter doctor
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Linux">
    <Steps>
      <Step title="Download Flutter">
        ```bash
        cd ~/development
        wget https://storage.googleapis.com/flutter_infra_release/releases/stable/linux/flutter_linux_3.16.0-stable.tar.xz
        tar xf flutter_linux_3.16.0-stable.tar.xz
        ```
      </Step>

      <Step title="Add to PATH">
        Add Flutter to your PATH in `~/.bashrc`:

        ```bash
        export PATH="$PATH:$HOME/development/flutter/bin"
        ```

        Reload:

        ```bash
        source ~/.bashrc
        ```
      </Step>

      <Step title="Install Dependencies">
        ```bash
        sudo apt-get update
        sudo apt-get install clang cmake ninja-build pkg-config libgtk-3-dev
        ```
      </Step>

      <Step title="Verify Installation">
        ```bash
        flutter doctor
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Supabase Project Setup

<Steps>

<Step title="Create Supabase Account">

1. Go to [supabase.com](https://supabase.com)
2. Click **Start your project**
3. Sign up with GitHub, Google, or email

</Step>

<Step title="Create a New Project">

1. Click **New Project**
2. Choose your organization
3. Enter project details:
   - **Name**: PhotoBoard
   - **Database Password**: Generate a secure password (save it!)
   - **Region**: Choose closest to your users
4. Click **Create new project**

<Note>
Project creation takes 1-2 minutes. Wait for the "Ready" status.
</Note>

</Step>

<Step title="Get API Credentials">

1. Navigate to **Settings** → **API**
2. Copy the following:
   - **Project URL** (e.g., `https://abcdefghijk.supabase.co`)
   - **anon public** key (not the service_role key!)

</Step>

<Step title="Set Up Database Schema">

Go to **SQL Editor** in your Supabase dashboard and run:

```sql
-- Create posts table
CREATE TABLE posts (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  image_url TEXT NOT NULL,
  comment TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Create index for faster queries
CREATE INDEX idx_posts_created_at ON posts(created_at DESC);

-- Enable Row Level Security
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;

-- Allow anyone to read posts
CREATE POLICY "Posts are publicly readable"
  ON posts FOR SELECT
  USING (true);

-- Allow anyone to insert posts
CREATE POLICY "Anyone can insert posts"
  ON posts FOR INSERT
  WITH CHECK (true);
```

Click **Run** to execute the SQL.

</Step>

<Step title="Configure Storage Bucket">

1. Navigate to **Storage** in the sidebar
2. Click **Create a new bucket**
3. Configure the bucket:
   - **Name**: `photos`
   - **Public bucket**: Toggle ON
   - **File size limit**: 5MB (recommended)
   - **Allowed MIME types**: `image/jpeg,image/png,image/webp`
4. Click **Create bucket**

Set storage policies in the **Policies** tab:

```sql
-- Allow public uploads to photos bucket
CREATE POLICY "Anyone can upload to photos bucket"
  ON storage.objects FOR INSERT
  WITH CHECK (bucket_id = 'photos');

-- Allow public read access
CREATE POLICY "Photos are publicly accessible"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'photos');
```

</Step>

</Steps>

## PhotoBoard Architecture

PhotoBoard follows **Clean Architecture** principles with clear separation of concerns:

```
lib/
├── main.dart                    # App initialization and Supabase setup
├── core/
│   └── services/
│       └── camera_service.dart  # Camera integration
└── features/
    └── posts/
        ├── data/
        │   └── repositories/
        │       └── post_repository.dart  # Supabase integration
        ├── domain/
        │   └── models/
        │       └── post.dart             # Post entity
        └── presentation/
            ├── providers/
            │   └── post_provider.dart    # State management
            ├── screens/                   # UI screens
            └── widgets/                   # Reusable components
```

### Key Components

<AccordionGroup>
  <Accordion title="PostRepository - Supabase Integration">
    The `PostRepository` handles all Supabase operations:

    ```dart post_repository.dart
    class PostRepository {
      final SupabaseClient _supabase;

      PostRepository({SupabaseClient? supabaseClient})
          : _supabase = supabaseClient ?? Supabase.instance.client;

      /// Upload image to Supabase Storage
      Future<String> uploadImage(File file, String fileName) async {
        final storagePath = 'public/$fileName';
        
        await _supabase.storage.from('photos').upload(storagePath, file);
        
        final String publicUrl = _supabase.storage
            .from('photos')
            .getPublicUrl(storagePath);
            
        return publicUrl;
      }

      /// Insert new post
      Future<void> createPost(String imageUrl, String comment) async {
        await _supabase.from('posts').insert({
          'image_url': imageUrl,
          'comment': comment,
        });
      }

      /// Fetch posts ordered by created_at
      Future<List<Post>> fetchPosts() async {
        final response = await _supabase
            .from('posts')
            .select()
            .order('created_at', ascending: false);
        
        return (response as List)
            .map((postMap) => Post.fromJson(postMap))
            .toList();
      }

      /// Real-time stream of posts
      Stream<List<Post>> fetchPostsStream() {
        return _supabase
            .from('posts')
            .stream(primaryKey: ['id'])
            .order('created_at', ascending: false)
            .map((maps) => maps.map((map) => Post.fromJson(map)).toList());
      }
    }
    ```
  </Accordion>

  <Accordion title="Post Model - Data Structure">
    ```dart post.dart
    class Post {
      final String id;
      final String imageUrl;
      final String comment;
      final DateTime createdAt;

      Post({
        required this.id,
        required this.imageUrl,
        required this.comment,
        required this.createdAt,
      });

      factory Post.fromJson(Map<String, dynamic> json) {
        return Post(
          id: json['id'].toString(),
          imageUrl: json['image_url'] as String,
          comment: json['comment'] as String,
          createdAt: DateTime.parse(json['created_at'] as String),
        );
      }
    }
    ```
  </Accordion>

  <Accordion title="App Initialization - main.dart">
    ```dart main.dart
    void main() async {
      WidgetsFlutterBinding.ensureInitialized();

      // Load environment variables
      await dotenv.load(fileName: "assets/.env");

      // Initialize Supabase
      await Supabase.initialize(
        url: dotenv.env['SUPABASE_URL'] ?? '',
        anonKey: dotenv.env['SUPABASE_ANON_KEY'] ?? '',
      );

      // Initialize app with Provider
      runApp(
        ChangeNotifierProvider(
          create: (context) => PostProvider(PostRepository()),
          child: const PhotoBoardApp(),
        ),
      );
    }
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Flutter doctor shows issues">
    Run `flutter doctor -v` for detailed diagnostics:

    ```bash
    flutter doctor -v
    ```

    Common fixes:
    - **Android licenses**: `flutter doctor --android-licenses`
    - **Missing cmdline-tools**: Install via Android Studio SDK Manager
    - **Xcode issues (macOS)**: Update to latest Xcode
  </Accordion>

  <Accordion title="Dependency conflicts">
    If you encounter dependency conflicts:

    ```bash
    flutter clean
    flutter pub cache repair
    flutter pub get
    ```
  </Accordion>

  <Accordion title="Supabase authentication errors">
    - Verify your **anon key** (not service_role key) in `.env`
    - Check that `SUPABASE_URL` has no trailing slash
    - Ensure Row Level Security policies are created
    - Confirm the storage bucket is set to **Public**
  </Accordion>

  <Accordion title="Build fails on Android">
    Check `android/app/build.gradle`:

    ```gradle
    android {
        compileSdkVersion 33  // or higher
        
        defaultConfig {
            minSdkVersion 21
            targetSdkVersion 33
        }
    }
    ```
  </Accordion>

  <Accordion title=".env file not loading">
    Ensure:
    1. File is at `assets/.env` (not root directory)
    2. `pubspec.yaml` includes it under assets
    3. Run `flutter pub get` after modifying pubspec.yaml
    4. Rebuild the app completely:
       ```bash
       flutter clean
       flutter run
       ```
  </Accordion>
</AccordionGroup>
