## Overview

The posts feed is the heart of PhotoBoard, displaying all participant contributions in real-time. Built with Supabase's streaming capabilities, the feed updates automatically when new posts are added by any participant.

## Real-Time Feed with Supabase Streams

The feed uses Supabase's `stream()` method to listen for database changes in real-time:

```dart lib/features/posts/data/repositories/post_repository.dart
Stream<List<Post>> fetchPostsStream() {
  return _supabase
      .from('posts')
      .stream(primaryKey: ['id'])
      .order('created_at', ascending: false)
      .map((maps) => maps.map((map) => Post.fromJson(map)).toList());
}
```

### How Streaming Works

<Steps>
  <Step title="Subscribe to Table">
    The app subscribes to the `posts` table with `stream(primaryKey: ['id'])`
  </Step>
  
  <Step title="Order Results">
    Posts are ordered by `created_at` in descending order (newest first)
  </Step>
  
  <Step title="Map to Models">
    Raw JSON data is mapped to `Post` objects using `Post.fromJson()`
  </Step>
  
  <Step title="Auto-Update UI">
    StreamBuilder automatically rebuilds the UI when new data arrives
  </Step>
</Steps>

<Note>
  Real-time updates happen automatically without requiring manual refresh. Users see new posts within seconds of publication.
</Note>

## Post Display with PostCard

Each post is rendered using the `PostCard` widget, which follows Instagram-style design:

```dart lib/features/posts/presentation/widgets/post_card.dart
class PostCard extends StatelessWidget {
  final Post post;

  const PostCard({super.key, required this.post});

  @override
  Widget build(BuildContext context) {
    final formattedDate = DateFormat('dd MMM yyyy').format(post.createdAt);

    return Card(
      margin: const EdgeInsets.symmetric(horizontal: 16, vertical: 12),
      elevation: 0,
      color: Colors.transparent,
      shape: RoundedRectangleBorder(
        borderRadius: BorderRadius.circular(20.0),
        side: BorderSide(color: Colors.grey[850]!, width: 1.0),
      ),
      child: Column(
        crossAxisAlignment: CrossAxisAlignment.stretch,
        children: [
          // Header with anonymous participant
          ListTile(
            leading: CircleAvatar(
              backgroundColor: Theme.of(context).colorScheme.primary.withOpacity(0.2),
              child: Icon(Icons.person_outline),
            ),
            title: const Text('Participante Anónimo'),
            trailing: Text(formattedDate),
          ),
          
          // Square image (1:1 aspect ratio)
          AspectRatio(
            aspectRatio: 1 / 1,
            child: Image.network(
              post.imageUrl,
              fit: BoxFit.cover,
            ),
          ),
          
          // Narrative/reflection
          Padding(
            padding: const EdgeInsets.all(16.0),
            child: Text(
              post.comment,
              style: const TextStyle(
                fontSize: 15,
                height: 1.6,
              ),
            ),
          ),
        ],
      ),
    );
  }
}
```

## Pull-to-Refresh Functionality

Users can manually refresh the feed by pulling down:

```dart lib/features/posts/presentation/screens/home_screen.dart
Future<void> _onRefresh() async {
  await Future.delayed(const Duration(seconds: 1));
}

// In build method:
return RefreshIndicator(
  onRefresh: _onRefresh,
  color: Theme.of(context).colorScheme.primary,
  backgroundColor: const Color(0xFF1E1E1E),
  child: ListView.builder(
    physics: const AlwaysScrollableScrollPhysics(),
    padding: const EdgeInsets.symmetric(vertical: 8, horizontal: 0),
    itemCount: posts.length,
    itemBuilder: (context, index) {
      final post = posts[index];
      return PostCard(post: post);
    },
  ),
);
```

<Note>
  While the stream already updates in real-time, pull-to-refresh provides a familiar gesture for users who expect manual control.
</Note>

## Post Data Model

Each post contains four essential fields:

```dart lib/features/posts/domain/models/post.dart
class Post {
  final String id;           // UUID from Supabase
  final String imageUrl;     // Public URL from Supabase Storage
  final String comment;      // Narrative reflection
  final DateTime createdAt;  // Timestamp for ordering

  Post({
    required this.id,
    required this.imageUrl,
    required this.comment,
    required this.createdAt,
  });

  factory Post.fromJson(Map<String, dynamic> json) {
    return Post(
      id: json['id'].toString(),
      imageUrl: json['image_url'] as String,
      comment: json['comment'] as String,
      createdAt: DateTime.parse(json['created_at'] as String),
    );
  }
}
```

### Field Descriptions

| Field | Type | Description |
|-------|------|-------------|
| `id` | String | Unique identifier (UUID) generated by Supabase |
| `imageUrl` | String | Public URL pointing to Supabase Storage bucket |
| `comment` | String | Participant's narrative reflection (SHOWED methodology) |
| `createdAt` | DateTime | UTC timestamp when post was created |

## Error Handling

The feed implements comprehensive error handling:

- **Image load failures**: Displays broken image icon
- **Empty states**: Encourages first contribution
- **Stream errors**: Caught and displayed to user
