Get PhotoBoard running on your device in just a few steps. This guide will take you from zero to a working PhotoVoice application that captures community narratives through photos and reflections.

## Prerequisites

Before you begin, ensure you have:

- Flutter SDK 3.0.0 or higher installed
- A Supabase account (free tier works perfectly)
- An Android device or emulator (iOS support coming soon)
- Git installed on your machine

<Steps>

<Step title="Clone the Repository">

Clone the PhotoBoard repository to your local machine:

```bash
git clone <repository-url>
cd photoboard
```

</Step>

<Step title="Install Dependencies">

Install all required Flutter packages:

```bash
flutter pub get
```

This will install the following key dependencies:

- `supabase_flutter: ^2.5.0` - Backend integration
- `image_picker: ^1.1.1` - Camera and gallery access
- `provider: ^6.1.2` - State management
- `flutter_dotenv: ^5.1.0` - Environment configuration
- `csv: ^6.0.0` - Data export for qualitative analysis
- `share_plus: ^7.2.1` - Content sharing capabilities

</Step>

<Step title="Configure Supabase">

Create a `.env` file in the `assets/` directory:

```bash
mkdir -p assets
touch assets/.env
```

Add your Supabase credentials to `assets/.env`:

```bash
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_ANON_KEY=your-anon-key
```

<Note>
You can find these credentials in your Supabase project dashboard under **Settings** → **API**.
</Note>

</Step>

<Step title="Set Up Supabase Database">

In your Supabase project dashboard, create the `posts` table:

```sql
CREATE TABLE posts (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  image_url TEXT NOT NULL,
  comment TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable Row Level Security
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;

-- Allow public read access
CREATE POLICY "Posts are publicly readable"
  ON posts FOR SELECT
  USING (true);

-- Allow public insert access
CREATE POLICY "Anyone can insert posts"
  ON posts FOR INSERT
  WITH CHECK (true);
```

</Step>

<Step title="Configure Storage Bucket">

Create a storage bucket for photos in your Supabase dashboard:

1. Navigate to **Storage** in your Supabase project
2. Click **New bucket**
3. Name it `photos`
4. Make it **Public**
5. Click **Create bucket**

Set up the storage policy:

```sql
-- Allow public uploads
CREATE POLICY "Anyone can upload photos"
  ON storage.objects FOR INSERT
  WITH CHECK (bucket_id = 'photos');

-- Allow public access to photos
CREATE POLICY "Photos are publicly accessible"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'photos');
```

</Step>

<Step title="Run the Application">

Connect your Android device or start an emulator, then run:

```bash
flutter run
```

For a production build:

```bash
flutter build apk --split-per-abi
```

The APK files will be generated in `build/app/outputs/flutter-apk/`.

</Step>

</Steps>

## Verify Your Setup

Once the app launches, you should see:

1. **Community Board**: A dark-themed interface showing the community photoboard
2. **Camera Button**: A circular button with a camera icon in the bottom-right corner
3. **Real-time Updates**: Posts appear instantly when shared by community members

## How PhotoBoard Initializes

Under the hood, PhotoBoard's `main.dart` performs the following initialization:

```dart main.dart
import 'package:flutter/material.dart';
import 'package:supabase_flutter/supabase_flutter.dart';
import 'package:flutter_dotenv/flutter_dotenv.dart';
import 'package:provider/provider.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  // Load environment variables from assets/.env
  await dotenv.load(fileName: "assets/.env");

  // Initialize Supabase with your credentials
  await Supabase.initialize(
    url: dotenv.env['SUPABASE_URL'] ?? '',
    anonKey: dotenv.env['SUPABASE_ANON_KEY'] ?? '',
  );

  // Set up Provider for state management
  runApp(
    ChangeNotifierProvider(
      create: (context) => PostProvider(PostRepository()),
      child: const PhotoBoardApp(),
    ),
  );
}
```

<Warning>
Never commit your `assets/.env` file to version control. Add it to `.gitignore` to protect your Supabase credentials.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: MissingPluginException">
    This usually means you need to rebuild the app after adding dependencies:
    
    ```bash
    flutter clean
    flutter pub get
    flutter run
    ```
  </Accordion>

  <Accordion title="Supabase connection fails">
    Verify your credentials in `assets/.env`:
    
    - Check that `SUPABASE_URL` doesn't have a trailing slash
    - Ensure `SUPABASE_ANON_KEY` is the anon/public key, not the service role key
    - Confirm the `.env` file is listed in `pubspec.yaml` under assets
  </Accordion>

  <Accordion title="Camera not working">
    Make sure you've granted camera permissions:
    
    - **Android**: Check `AndroidManifest.xml` includes camera permissions
    - **Physical Device**: Grant camera permission when prompted
    - **Emulator**: Use a virtual camera in AVD settings
  </Accordion>
</AccordionGroup>
